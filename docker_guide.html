Dockerization Guide: Document Approval System

Welcome! This guide will take you through the levels of dockerizing your React + Spring Boot application. We will start from scratch and build up to a full Docker Compose setup.

Level 1: Initial Setup (Pre-Docker)

Before we containerize, let's understand what we are running. You currently have:

  • Backend: Spring Boot (Java 17+)
  • Frontend: React (Vite)
  • Database: PostgreSQL

Ensure you have Docker Desktop installed and running.

Level 2: The Backend Dockerfile

We need to create a blueprint for our backend container. We'll use a "Multi-Stage Build" to keep the image small.

Action: Create a file named Dockerfile in the root directory of your project (where pom.xml is).

File: Dockerfile 
# Stage 1: Build the application
FROM maven:3.8.5-openjdk-17 AS build
WORKDIR /app
COPY pom.xml .
COPY src ./src
# Build the jar, skipping tests to save time during guide
RUN mvn clean package -DskipTests

# Stage 2: Run the application
FROM eclipse-temurin:17-jdk-alpine
WORKDIR /app
# Copy the built jar from the previous stage
COPY --from=build /app/target/*.jar app.jar

# Expose the port Spring Boot runs on
EXPOSE 8080

# Command to run the application
ENTRYPOINT ["java", "-jar", "app.jar"]

Level 3: The Frontend Dockerfile

Now for the React frontend. We will build the static files and serve them using Nginx, a high-performance web server.

Action: Create a file named Dockerfile in the frontend directory (frontend/Dockerfile).

File: frontend/Dockerfile 
# Stage 1: Build the React app
FROM node:18-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

# Stage 2: Serve with Nginx
FROM nginx:alpine
# Copy the build output to Nginx's html directory
COPY --from=build /app/dist /usr/share/nginx/html
# Expose port 80 (standard web port)
EXPOSE 80
# Start Nginx
CMD ["nginx", "-g", "daemon off;"]

Level 4: Volumes & Networking (The Concepts)

Volumes: Containers are ephemeral (temporary). If you delete a container, its data is gone. We use Volumes to save data (like your database and uploaded files) on your host machine so it survives restarts.

Networking: By default, containers are isolated. We need a Network so the Backend can talk to the Database, and the Frontend can talk to the Backend.

We will implement both of these in the next step using Docker Compose.

Level 5: Docker Compose (Step-by-Step)

Docker Compose allows us to define all our services, networks, and volumes in a single file. We will build this file piece by piece so you understand exactly what is happening.

Action: Create a file named docker-compose.yml in the root directory.

Step 5.1: The Skeleton

First, we define the version and create the main blocks.

version: '3.8'

services:
  # We will add our services here...

volumes:
  # We will add volumes here...

networks:
  # We will add networks here...

Step 5.2: The Database Service

We add the PostgreSQL database. We set the username, password, and database name using environment variables. We also mount a volume so data persists.

services:
  db:
    image: postgres:15
    container_name: approval_db
    environment:
      POSTGRES_DB: document_approval_db
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: 00000
    volumes:
      - db_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    networks:
      - app-network

Step 5.3: The Backend Service

This builds the Dockerfile in the current directory (.). Crucially, we override the SPRING_DATASOURCE_URL to point to the db service we just defined (instead of localhost).

  backend:
    build: .
    container_name: approval_backend
    depends_on:
      - db
    environment:
      # 'db' here refers to the service name defined above
      SPRING_DATASOURCE_URL: jdbc:postgresql://db:5432/document_approval_db
      SPRING_DATASOURCE_USERNAME: postgres
      SPRING_DATASOURCE_PASSWORD: 00000
    volumes:
      # Map the uploads directory to the host so files aren't lost
      - ./uploads:/app/uploads
    ports:
      - "8080:8080"
    networks:
      - app-network

Step 5.4: The Frontend Service

This builds the Dockerfile in the ./frontend directory. It maps port 3000 on your machine to port 80 inside the container (where Nginx is listening).

  frontend:
    build: ./frontend
    container_name: approval_frontend
    depends_on:
      - backend
    ports:
      - "3000:80"
    networks:
      - app-network

Step 5.5: Volumes & Networks

Finally, at the bottom of the file, we explicitly define the volume and network we referenced above.

volumes:
  db_data:

networks:
  app-network:
    driver: bridge

The Complete File

Here is the full docker-compose.yml file for you to copy:

version: '3.8'

  backend:

Level 6: Launch!

Now that everything is set up, open your terminal in the root directory and run:

docker-compose up --build

What to expect:

  1. Docker will download the images (Postgres, Java, Node, Nginx).
  2. It will build your Spring Boot app.
  3. It will build your React app.
  4. It will start the Database, then the Backend, then the Frontend.
Access your app:
Frontend: http://localhost:3000
Backend API: http://localhost:8080/api

To stop everything, press Ctrl+C or run docker-compose down.
